# Phase 4: IMPLEMENT INVITE EMAIL DELIVERY - Execution Plan

## 1. Setup Email Configuration
- **File:** `backend/syntra/settings.py`
- **Action:** Add email settings configured via `.env` variables.
- **Details:** 
  - Read `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD`, and `EMAIL_USE_TLS`.
  - Default `EMAIL_BACKEND` to `django.core.mail.backends.smtp.EmailBackend` if `EMAIL_HOST` is present, otherwise `django.core.mail.backends.console.EmailBackend`.

## 2. Create Celery Task for Emails
- **File:** `backend/organizer/tasks.py`
- **Action:** Add a new Celery task `send_coordinator_invite_email`.
- **Details:**
  - Accepts `email`, `hackathon_name`, and `invite_url`.
  - Uses `render_to_string` to compile an HTML template.
  - Uses `EmailMultiAlternatives` to send the multipart email (HTML + Plain text fallback).
  - Handles SMTP errors and retries gracefully.

## 3. Create Email Template
- **File:** `frontend/templates/emails/coordinator_invite.html`
- **Action:** Create a responsive, well-designed HTML email template.
- **Details:** Ensure it matches Syntra's branding and includes a clear call to action to accept the invitation.

## 4. Refactor API View
- **File:** `backend/organizer/api_views.py`
- **Action:** Update `HackathonViewSet.assign_coordinator`.
- **Details:** Replace the synchronous `send_mail` call with `send_coordinator_invite_email.delay(...)` to queue the invite asynchronously.

## 5. Write Tests
- **File:** `backend/organizer/tests.py`
- **Action:** Add/Update tests.
- **Details:**
  - `test_celery_task_sends_email`: Test the standalone task execution.
  - `test_assign_new_user_creates_invite`: Update to patch the Celery task `.delay` method instead of `send_mail`.
