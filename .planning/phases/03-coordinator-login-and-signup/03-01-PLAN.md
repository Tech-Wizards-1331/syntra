# Phase 3: Coordinator Login and Signup Plan

## 1. Backend Implementation: Invite Dispatch

**Task:** Modify the `assign_coordinator` endpoint to handle unknown emails by creating a shell user and sending an invite link.
<read_first>
- `backend/organizer/api_views.py`
- `backend/accounts/models.py`
</read_first>
<action>
Modify `assign_coordinator` in `backend/organizer/api_views.py`.
When `User.objects.get(email=email)` raises `User.DoesNotExist`:
1. Create a new User: `user = User.objects.create(email=email, role=User.Role.COORDINATOR)`.
2. Set unusable password: `user.set_unusable_password()` and save.
3. Generate a token using Django's `PasswordResetTokenGenerator`:
   ```python
   from django.contrib.auth.tokens import default_token_generator
   from django.utils.http import urlsafe_base64_encode
   from django.utils.encoding import force_bytes
   uid = urlsafe_base64_encode(force_bytes(user.pk))
   token = default_token_generator.make_token(user)
   ```
4. Build the invite URL using `request.build_absolute_uri`: `/accounts/invite/accept/{uid}/{token}/`.
5. Send an email to the user using Django's `send_mail`:
   - Subject: "You have been invited as a Coordinator"
   - Message: Include the generated invite link.
   - Use a try/except block to catch email sending errors and log them, so the API doesn't fail if SMTP isn't configured.
</action>
<acceptance_criteria>
- `backend/organizer/api_views.py` contains `default_token_generator.make_token(user)`
- `backend/organizer/api_views.py` contains `send_mail`
- Assigning a new email returns 201 Created instead of 404 Not Found.
</acceptance_criteria>

## 2. Backend Implementation: Invite Acceptance Logic

**Task:** Create the URL, View, and Form for the coordinator to accept the invitation and set their password.
<read_first>
- `backend/accounts/urls.py`
- `backend/accounts/views.py`
- `backend/accounts/forms.py`
</read_first>
<action>
1. **Forms**: In `backend/accounts/forms.py`, create `class AcceptInviteForm(forms.ModelForm)` with fields `full_name`, `password1`, and `password2`. Implement `clean()` to ensure passwords match. Use `INPUT_CLASS` for styling exactly like `SignUpForm`.
2. **Views**: In `backend/accounts/views.py`, create `accept_invite_view(request, uidb64, token)`.
   - Use `urlsafe_base64_decode` to get the user ID.
   - Fetch the User. If it fails, `messages.error(request, 'Invalid invite link.')` and redirect to `login`.
   - Verify token: `default_token_generator.check_token(user, token)`. If false, redirect to login with error.
   - If valid, process `AcceptInviteForm`. On POST valid: `user.full_name = form.cleaned_data['full_name']`, `user.set_password(form.cleaned_data['password1'])`, `user.save()`.
   - Log the user in with `login(request, user, backend='django.contrib.auth.backends.ModelBackend')`.
   - Redirect to dashboard via `_redirect_by_state(user)`.
3. **URLs**: In `backend/accounts/urls.py`, add `path('invite/accept/<uidb64>/<token>/', views.accept_invite_view, name='accept_invite')`.
</action>
<acceptance_criteria>
- `backend/accounts/urls.py` contains `path('invite/accept/<uidb64>/<token>/'`
- `backend/accounts/views.py` contains `def accept_invite_view`
